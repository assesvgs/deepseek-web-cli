# chat.ts 完整手动验收方案

## 前置说明

### 测试环境

本方案的测试环境是 `Test/` 目录下的三个 fixture 子目录：

| 目录 | system.md (局部) | system-all.md (全局) | think.md | tool.md | test.txt |
|------|------------------|---------------------|----------|---------|----------|
| test11 | 英文助手（用英文回复） | 中文助手（用中文回复） | 4步结构化推理 | 完整工具说明 (150行) | 嘉豪哲学文本（1行） |
| test22 | 中文助手（用中文回复） | 英文助手（用英文回复） | 4步中文推理 | 完整工具说明 (150行) | BiliBili 广告账号列表（68行） |
| test33 | **不存在** | 猫娘角色扮演（长文本） | 4步结构化推理 | 完整工具说明 (150行) | Linux 内核漏洞告警（12行） |

**设计意图**：
- test11 与 test22 的 system.md / system-all.md **内容互换**，用于验证局部优先策略是否按目录独立生效
- test11 与 test22 的 think.md **内容不同**（英文 4 步 vs 中文 4 步），用于验证思考提示词是否随目录切换
- test33 **没有 system.md**，只有 system-all.md，用于验证无局部提示词时是否正确回退到全局
- 三个目录的 **tool.md 完全相同**，用于验证工具提示词是否稳定
- 三个目录的 **test.txt 内容完全不同**（行数差异大），用于验证 read/edit/write/upload 在不同文件上的行为

### 已知限制（2026-05-14）

DeepSeek 官方已关闭专家模式 (pro/expert) 的文件上传功能。因此 **`/model pro` + `/upload` 组合不测试**。flash 和 vision 模式的上传仍需测试。

### 不覆盖项

| 项目 | 原因 |
|------|------|
| `/reauth` | 需要 Chrome 浏览器 + 手动登录，无法自动化 |
| 401 自动重登 | 依赖真实过期 token，无法构造 |
| PoW 求解 | 依赖 DeepSeek 返回的加密挑战，交互测试中间接覆盖 |

### 运行方式

```bash
# 确保凭证就绪
./chat.ts

# 在 REPL 中按本方案逐条执行
```

### 约定

- `>` 表示在 REPL 提示符下输入
- `←` 表示期望看到的输出或行为
- 所有路径均相对于 `Test/` 目录，例如 `test11/test.txt`
- "发送消息" 指输入任意非命令文本让 AI 回复，用于触发首轮提示词注入
